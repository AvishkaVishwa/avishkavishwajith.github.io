How I fixed ESP32 brownout issues

Brownout happens when voltage dips during Wi-Fi or motor load peaks. Here’s a practical fix list.

ESP32PowerDebug

What is a brownout?

A brownout reset happens when the supply voltage drops below the safe level. The ESP32 detects it and resets to protect memory and stability.

Common causes (real-world)

• Weak power source (USB port, cheap adapter)
• Regulator can’t handle peak current
• Long/thin wires → voltage drop
• Motors causing noise + dips on the same supply
• Bad grounding / shared return paths

Quick fixes (recommended order)

1) Add bulk capacitor near ESP32 (470µF–1000µF) + 0.1µF ceramic
2) Use a better 5V regulator with enough peak current
3) Keep power wires short and thicker
4) Separate motor power and logic power (common ground only)
5) Add proper decoupling on motor driver + snubbers if needed

My typical solution

In my robot projects, brownout happened when Wi-Fi AP + web server started while motors were running. Adding a 470µF capacitor near the ESP32-S3 and improving the power supply stability solved it.
