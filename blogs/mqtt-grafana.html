MQTT + Grafana dashboard setup (beginner friendly)

How sensor data flows: device → MQTT → database → Grafana dashboards and alerts.

MQTTGrafanaIoT

Basic flow

1) ESP32 publishes data to MQTT topics (example: factory/room1/temp)
2) A bridge writes that data to a time-series database (InfluxDB, Timescale, etc.)
3) Grafana reads the database and shows charts + alerts

Good topic design

Use a consistent structure:
site/device/sensor → example: farm/node01/humidity
Keep payloads JSON if possible (easy to extend).

Alerts

Grafana alerts work best when your data is clean and you define thresholds clearly (temperature too high, humidity too low, smoke detected, etc.).
